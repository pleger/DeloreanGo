# Timepoint Web IDE

This folder contains a browser-based IDE to:

- run JavaScript code,
- create timepoints (explicit or implicit),
- inspect and edit variables in a selected timepoint,
- resume execution from that timepoint,
- create a new timeline branch when edited variables change execution context.

## Run locally

Option A (simple): open directly:

- `web_ide_timepoint/index.html`

Option B (recommended) with a local server:

```bash
cd web_ide_timepoint
python3 -m http.server 8080
```

Open: `http://localhost:8080`

## Usage flow

1. Write or paste code in **Panel 1 (Source code)**.
2. Create explicit timepoints by clicking line numbers in the gutter.
3. In **Panel 6 (Settings)** choose mode:
   - `Explicit`: only marked lines create timepoints.
   - `Implicit`: each executed line can create a timepoint.
4. Click **Run**.
5. Click a node in **Panel 4 (Timeline)**.
6. Edit one or more variables in **Panel 5 (Timepoint variables)** and click **Save edits**.
7. Click **Resume from TP**.

## Resume behavior

- If no variable edits were saved, **Resume from TP** replays the selected timeline from that point.
- If variable edits were saved, **Resume from TP** runs a **branched execution** and creates a new timeline row.
- The new timeline keeps a reference to its parent point and becomes the active timeline.

## Panels

- **Panel 1**: code editor with clickable gutter.
- **Panel 2**: console output and errors.
- **Panel 3**: watch variables (add/remove).
- **Panel 4**: timeline view with root runs and branch timelines.
- **Panel 5**: editable variable snapshot for selected timepoint.
- **Panel 6**: mode, speed, and execution controls.

## Internal behavior

- Source code is transformed in-browser to inject per-line hooks.
- Hooks create snapshots (`line`, `time`, `name`, `snapshot`) for timepoints.
- Edited values are stored per selected timepoint.
- Branch runs are generated by re-executing and switching context at the selected timepoint.

## UX notes

- The gutter only shows icons for explicit timepoints.
- Implicit timepoints are represented in the timeline, not by marking every line.

## Current limitations

- This is a lightweight browser demo, not a full debugger.
- Code is executed as instrumented JavaScript in the client.
- Some complex syntax patterns may require additional transformer updates.
